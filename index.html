LIVE EXECUTION ENGINE

Zerodha Trading Integration

Architecture of an automated algorithmic bridging layer that reconciles offline quantitative signals with live brokerage portfolios via the Zerodha KiteConnect API.

1. KiteConnect Integration

A direct OAuth gateway to the Zerodha Kite API utilizing the kiteconnect Python client. It continuously authenticates and fetches live market status, holdings, and intraday positions arrays.

2. Live State Reconciliation

Matches real-time traded instrument tokens against offline machine learning output stored in trading_signals.csv. The engine calculates realized and unrealized P/L by reconciling API-held quantities against locally derived target exposures.

3. Real-Time Price Polling

Executes lightweight JSON requests to kite.quote() to fetch real-time bid/ask snapshots for dynamic mark-to-market valuations, bypassing slow historical data endpoints when live trading is active.

4. Fallback Data Pipeline

Implements an offline fallback mechanism: if the KiteConnect API rate limits are hit or the live ticker socket drops, the system gracefully falls back to querying the latest Yahoo Finance datasets to estimate net asset value.
